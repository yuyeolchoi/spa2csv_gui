# WORK_ORDER — SPA to CSV 변환 툴 (v0.1)

## 배경
Thermo Fisher OMNIC에서 IR(FT-IR) 분석 시 생성되는 `.spa` 바이너리 스펙트럼 파일을,
in-situ 측정 후 수십~수백 개를 일일이 OMNIC에서 열어 CSV로 내보내는 작업이 반복적이고 비효율적임.
SPA 파일을 일괄 로드하여 CSV로 자동 변환하는 데스크톱 GUI 툴을 만든다.

## 작업 디렉토리
`F:\Code\SPA_to_CSV` (이 폴더에서만 작업. git 저장소로 초기화할 것)

## 기술 스택
- Python 3.11+, GUI는 **tkinter** (외부 GUI 프레임워크 금지)
- 수치 처리: numpy (필요 시)
- 패키징: 단일 진입점 `spa_to_csv/` 패키지 + `main.py` 실행 스크립트
- 테스트: pytest (`tests/` 폴더, 마커 없이 기본 실행으로 통과해야 함)

## 기능 요구사항

### 1. SPA 파서 (`spa_to_csv/parser.py`)
- OMNIC `.spa` 바이너리 포맷을 직접 파싱한다 (OMNIC 설치 불필요).
  - 공개된 SPA 포맷 리버스엔지니어링 자료 기반으로 구현할 것
    (spectrochempy의 `read_omnic`, GitHub의 여러 spa reader 구현 참고 가능한 수준의 로직).
  - 핵심: 파일 내 디렉토리 엔트리를 스캔하여 스펙트럼 데이터 블록(float32 배열)과
    헤더(포인트 수, 시작/끝 파수 wavenumber)를 찾고, x축(파수)과 y축(흡광도/투과율)을 복원.
- 반환: `(wavenumbers: np.ndarray, intensities: np.ndarray, meta: dict)`
- 파싱 실패 시 명확한 예외(`SpaParseError`)를 발생시키고 파일명을 포함한 메시지 제공.

### 2. CSV 변환 (`spa_to_csv/converter.py`)
- 출력 CSV: 2열 (`Wavenumber (cm-1)`, `Intensity`), 헤더 1행, UTF-8, 소수점 표기 그대로.
- 출력 파일명은 입력과 동일 (`sample01.spa` → `sample01.csv`), 기본 저장 위치는 원본 파일과 같은 폴더.
- 동일 이름 CSV가 이미 있으면 덮어쓴다 (v0.1 정책, 추후 옵션화 가능).

### 3. GUI (`spa_to_csv/app.py`)
레이아웃 (3단 구성):
```
+----------------+   +--------+   +----------------+
|  SPA 파일 리스트 |   | 파일 로드 |   |  CSV 결과 리스트 |
|  (Listbox,     |   | 폴더 로드 |   |  (Listbox)     |
|   스크롤바)     |   | 작업 시작 |   |                |
|                |   | (목록 비우기)|  |                |
+----------------+   +--------+   +----------------+
|                    상태바 (진행 상황 표시)            |
+---------------------------------------------------+
```
- **파일 로드**: 파일 다이얼로그(다중 선택, `*.spa` 필터) → 왼쪽 리스트에 추가 (중복 제외)
- **폴더 로드**: 폴더 다이얼로그 → 해당 폴더의 `*.spa` 전부 왼쪽 리스트에 추가
- **작업 시작**: 왼쪽 리스트의 모든 파일을 순차 변환.
  - 변환 성공 → 오른쪽 리스트에 생성된 CSV 파일명 추가
  - 변환 실패 → 오른쪽 리스트에 `[실패] 파일명 — 사유` 형태로 표시하고 다음 파일 계속 진행
  - 변환 중 GUI가 얼지 않도록 스레드 사용, 상태바에 `n/전체` 진행 표시
- 창 제목: `SPA to CSV Converter`
- 리스트 항목은 파일명만 표시하되 전체 경로는 내부에 보관 (툴팁 불필요)

### 4. 테스트 (`tests/`)
- 파서: 합성 SPA 바이너리 픽스처를 코드로 생성하여 라운드트립 검증
  (실제 SPA 파일이 없어도 테스트 가능해야 함).
- 변환기: 임시 폴더에 CSV 생성 → 내용/파일명 검증.
- GUI 로직: 파일 목록 추가/중복 제거 등 tkinter 인스턴스 없이 검증 가능한 순수 로직은 분리하여 테스트.
- GUI 자체 스모크 테스트는 생략 가능 (headless 환경 고려).
